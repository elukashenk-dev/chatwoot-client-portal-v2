# B2B Product Goal

## Назначение

`chatwoot-client-portal-v2` нужен для B2B-услуги: установить и настроить Chatwoot для компании, а сверху дать отдельное брендированное PWA-приложение для клиентов этой компании.

Это не один общий чат для собственных клиентов владельца проекта. Это переиспользуемый клиентский слой, который можно брендировать и разворачивать под разные B2B-компании.

## Роли

- Поставщик услуги:
  владелец этого предложения. Продает установку, deployment, кастомизацию и развитие системы поддержки.
- B2B-компания:
  прямой покупатель услуги. Примеры: сервисная компания, бухгалтерская фирма, юридическая компания, SaaS, агентство, клиника, образовательный центр, локальный сервис или управляющая компания.
- Конечный клиент:
  клиент B2B-компании. Устанавливает PWA-чат через браузер и пишет компании.

## Продуктовая Цепочка

1. Поставщик услуги продает B2B-компании установку Chatwoot и клиентский PWA-чат.
2. B2B-компания получает операторскую систему поддержки на базе Chatwoot.
3. Клиенты B2B-компании устанавливают брендированный PWA-чат.
4. Конечные клиенты пишут компании через PWA.
5. Операторы компании отвечают из Chatwoot.

## Что Продается

Предложение - это комплект, а не просто UI чата:

- Chatwoot для бизнеса:
  операторская панель, inbox, агенты, команды, статусы, история переписок, метки и автоматизации.
- Клиентское приложение:
  отдельный PWA-чат со своим UI, авторизацией, брендингом компании и installable app experience.
- Внедрение:
  установка, настройка, домен, SSL, email, при необходимости Telegram/WhatsApp, права доступа и базовые сценарии поддержки.
- Кастомизация:
  логотип, цвета, тексты, экраны входа, экран установки, профиль компании и возможные интеграции с CRM, 1C, сайтом или API.

## Ключевая Ценность

B2B-компания получает собственный брендированный канал общения с клиентами. Он не зависит полностью от WhatsApp, Telegram или email, при этом операторы работают внутри удобной системы поддержки Chatwoot.

## Архитектурный Смысл

PWA-чат - это дополнительная клиентская оболочка для конечных клиентов B2B-компании. Он не заменяет Chatwoot.

Chatwoot остается операторской системой и system of record для чата. Portal backend остается authority-boundary между клиентским PWA и Chatwoot.

Брендинг, тексты, экраны входа, install experience, профиль компании и будущая клиентская кастомизация должны считаться полноценными продуктовыми областями портала, а не разовыми визуальными правками внутри chat feature.
