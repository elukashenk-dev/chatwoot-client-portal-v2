Проанализируй текущий проект и подготовь production-ready реализацию клиентского портала в связке Tailwind + Preline на основе уже утверждённых экранов.

Цель:
Перенести существующие HTML-прототипы экранов в реальную архитектуру проекта без CDN и без одноразовой вёрстки. Нужен аккуратный, поддерживаемый, production-ready UI.

Исходные экраны:
1. Экран входа
2. Экран подтверждения кода
3. Экран нового аккаунта
4. Экран создания пароля
5. Экран чата

Что нужно сделать:
1. Сначала проанализируй текущий стек проекта и выбери правильную интеграцию Tailwind + Preline под него.
   - Если это Next.js / React / Vue / Vite / Laravel / другой стек, адаптируй решение под текущую архитектуру.
   - Не ломай существующую структуру проекта без необходимости.

2. Убери прототипный подход.
   - Не использовать Tailwind Play CDN.
   - Подключить Tailwind локально через нормальную конфигурацию проекта.
   - Подключить Preline правильно для реального проекта.
   - Вынести повторяющиеся стили в theme tokens / config / shared utilities.

3. Построй UI как систему компонентов, а не как набор отдельных страниц.
   Минимально выделить:
   - AppShell / AuthShell
   - PortalHeader / ChatHeader
   - ServiceFooter
   - FormField
   - PasswordField с show/hide
   - OtpInputGroup
   - StatusBadge
   - MessageBubbleIncoming
   - MessageBubbleOutgoing
   - ReplyPreview
   - AttachmentCard
   - QuickEmojiBar
   - MessageComposer
   - MessageCalendarPopover

4. Соблюсти уже утверждённый визуальный стиль.
   Стиль:
   - строгий B2B
   - светлый фон
   - тёмно-синий brand color
   - мягкие тонкие декоративные уголки
   - лёгкий dot-grid
   - iPad portrait / desktop app window ощущение
   - mobile-first адаптация без поломок

5. Реализовать адаптивность.
   - Mobile-first
   - Хороший вид на телефоне
   - Хороший вид на iPad portrait
   - Хороший вид в desktop app window примерно 750x1060
   - Не допускать ломания header / footer / composer / emoji bar / calendar popover

6. Для чата реализовать UI-логику на уровне интерфейса.
   - Показываются последние 20 сообщений
   - Кнопка загрузки более ранних сообщений
   - Календарь сообщений вместо кнопки нового обращения
   - Popover календаря
   - Группировка сообщений блоками
   - unread divider
   - reply state
   - attachment card
   - quick emoji bar с горизонтальным скроллом на mobile
   - composer со скрепкой, голосовым сообщением и отправкой

7. Для auth-экранов реализовать:
   - валидацию
   - loading states
   - error states
   - success states
   - OTP-поля
   - password rules
   - show/hide password

8. Сделай код готовым к дальнейшей интеграции с backend.
   - Пока без реального API, но с понятными props / state / interfaces / types
   - Подготовь места для интеграции с Chatwoot backend
   - Не хардкодь бизнес-логику глубоко в шаблоны

9. Наведи порядок в структуре файлов.
   - Раздели pages/screens, shared ui, chat components, auth components, utils, constants
   - Удали дублирование
   - Сделай понятные имена файлов

10. В конце дай результат в таком формате:
   - что именно изменено
   - какие новые файлы созданы
   - какие зависимости добавлены
   - как запускать
   - какие места оставлены под backend интеграцию
   - какие UI-решения ты принял

Требования к качеству:
- Не делать “быстрый прототип”
- Не оставлять CDN, inline-хаки и временные решения
- Не перепридумывать стиль
- Не ухудшать текущий визуальный язык
- Все изменения должны быть аккуратными и согласованными между экранами

Режим работы:
Сначала изучи проект и предложи короткий план миграции.
Потом сам выполни реализацию.
Если есть неоднозначность, принимай самое консервативное и поддерживаемое решение.

Теперь проведи второй проход как senior frontend engineer и product UI engineer.

Проверь результат по следующим пунктам:
1. Нет ли прототипного кода, временных решений и inline-хака
2. Везде ли корректно подключён и используется Tailwind + Preline
3. Нет ли дублирующихся компонентов или повторяющихся классов
4. Нет ли проблем в mobile layout
5. Нет ли проблем в iPad / desktop app window layout
6. Нет ли визуальных расхождений между auth-экранами и экраном чата
7. Нет ли accessibility-проблем:
   - labels
   - aria
   - focus states
   - keyboard navigation
8. Нет ли проблем с popover календаря, composer, reply state, emoji scroll
9. Нет ли лишних зависимостей
10. Где нужно, добавь финальную полировку UI

После этого:
- внеси исправления сам
- покажи итоговый список улучшений
- перечисли, что ещё осталось только для backend-интеграции

Теперь подготовь фронтенд к интеграции с backend и Chatwoot.

Нужно:
1. Выделить data layer / service layer для:
   - auth
   - OTP verify
   - register request
   - password setup
   - chat history
   - load older messages
   - load messages by date
   - send message
   - attachments

2. Создать интерфейсы / типы / контракты данных
3. Подготовить mock services и API placeholders
4. Не подключать реальный backend, но сделать архитектуру готовой к нему
5. Все UI-компоненты перевести на нормальные props / state / types
6. Показать, какие endpoints потом нужно будет подключить

не делай всё в одном giant-file
не используй CDN
не оставляй demo-скрипты
не делай “примерные” компоненты без структуры
сохрани визуальный стиль утверждённых экранов
